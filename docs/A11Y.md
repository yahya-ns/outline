# Accessibility checklist

## Prerequisites

This checklist assumes familiarity with the [FRONTEND.md](FRONTEND.md) cross-cutting patterns (especially error handling, modal/dialog management, and the theme system) and the [EDITOR.md](EDITOR.md) ProseMirror node/mark model. The automated scan below runs against the running Vite dev server, so set up your local environment per [DEVELOPMENT.md](DEVELOPMENT.md) before running it.

This document captures the accessibility concerns specific to Outline. Use it when reviewing frontend changes that touch the editor, the sidebar, modals, forms, or navigation. The automated scan lives in the [A11y workflow](../.github/workflows/a11y.yml) and the `yarn a11y` script.

See [FRONTEND.md](FRONTEND.md) and [EDITOR.md](EDITOR.md) for the broader React and ProseMirror context.

## Skip navigation

`app/components/SkipNavLink.tsx` and `app/components/SkipNavContent.tsx` provide the skip-to-content link that becomes visible on focus. Every authenticated scene must keep both mounted in the layout; do not remove them when refactoring `AuthenticatedLayout`.

## Keyboard

- The editor host (`app/editor/index.tsx`) sets `aria-label="Editor content"` on the view DOM. Do not remove it.
- The `PreventTab` extension (`app/editor/extensions/PreventTab.ts`) stops the browser from trapping Tab inside the editor. Preserve its keymap when adding new editor bindings.
- The shortcut router (`app/hooks/useKeyDown.ts`) is IME-aware: it tracks an `imeOpen` flag via `compositionstart` / `compositionend` listeners (`useKeyDown.ts:23,87-93`) and bails at the top-level handler when composition is in progress. Preserve this guard when adding shortcuts.
- Sidebar disclosures (`app/components/Sidebar/components/Disclosure.tsx`) use `aria-expanded` and `aria-label`. Tree-node context menus must remain keyboard-reachable; do not gate them on `onClick` only.
- The Radix `Modal` (`app/components/Modal.tsx`) traps focus and pops on `Escape`. The modal stack (`app/components/Dialogs.tsx`) reads from `DialogsStore.modalStack`; only the top modal should trap focus.
- `app/components/primitives/Drawer.tsx` (vaul) needs its own focus management — verify popovers and drawers do not leak focus to the page.

## ARIA

- The active nav link gets `aria-current` from `app/components/Sidebar/components/NavLink.tsx`.
- The header aside uses `role="complementary"` in `app/components/Sidebar/Aside.tsx`.
- The table primitive (`app/components/Table.tsx`) sets `role="table"`, `"row"`, `"cell"`. Custom tables outside this primitive must set the same roles.
- The notifications popover uses `aria-describedby={undefined}` on the trigger to suppress the auto-generated label. Preserve this pattern on new popovers.
- The `comment` mark on selected text carries an `id` and `userId`; the comment thread sidebar reads these. Custom selection-based marks must include the same shape.

## Color and contrast

- Theme tokens live in `shared/styles/theme.ts` (`buildLightTheme`, `buildDarkTheme`, `buildPitchBlackTheme`). Every new color must be added to all three palettes.
- `prefers-color-scheme` drives the resolved theme. Do not hardcode color values outside the token set.
- Pitch-black is an explicit accessibility option — verify new surfaces look right under all three palettes.

## Forms

- `app/components/primitives/Form.tsx` auto-wires the CSRF hidden input (the only thing it does automatically — 19 lines total). It does **not** wire label associations or `aria-invalid`; for those, you must use `<label htmlFor>` and `aria-invalid` / `aria-describedby` yourself.
- Custom forms must set `htmlFor` on every `<label>` linking to the corresponding input's `id`, and set `aria-invalid` + `aria-describedby` (pointing to the error message element) when validation fails.
- The OTP input uses `@radix-ui/react-one-time-password-field` — let Radix manage focus between segments.

## Toasts and announcements

- Toasts use `sonner` (`<Toaster>` in the provider tree). Sonner announces via `aria-live="polite"`; preserve the default behaviour.
- Long-running operations (exports, imports) announce progress through `UiStore.exportToasts`. Do not bypass it with a manual toast.

## Images and media

- The `Image` node (`shared/editor/nodes/Image.tsx`) requires an `alt` attribute. Empty `alt` is intentional for decorative images; otherwise the `alt` must describe the image.
- Embeds (`shared/editor/embeds/`) render provider components. Each provider must produce a `title` attribute on its outer element.
- Avatar images should keep `alt=""` for decorative use; user names appear in the surrounding `aria-label`.

## Touchpoints for review

When reviewing a PR, check the diff for:

- New `Button` instances missing `aria-label` for icon-only variants.
- New `Modal` / `Popover` / `Drawer` mounts that do not go through `DialogsStore` or `primitives/`.
- New color values that bypass the theme tokens.
- New images or embeds without `alt` or `title`.
- New keyboard shortcuts without IME guards.
- Removal of any `aria-*` attribute on existing elements.

## Automated checks

`yarn a11y` runs `@axe-core/cli` against the dev server on port 3001. The CI workflow at `.github/workflows/a11y.yml` triggers on every pull request, uploads the report as an artifact, and fails the job on violations. The `@axe-core/cli` dev-dep must be added to `devDependencies` before the script will run — see the workflow file for the note.

To run locally: `yarn vite:dev` in one terminal, `yarn a11y` in another.

### What this scan actually covers (and doesn't)

The workflow boots the Vite dev server with no backend, no auth, no DB, and runs axe against the single URL passed in. In that configuration the SPA renders the unauthenticated login page; axe scans that surface. It does **not** exercise the authenticated app — the editor, the sidebar tree, the collection/document pages, the settings scenes, the dialogs, or the modals.

For full app coverage, drive axe via a Playwright/Cypress integration that signs in as a test user and walks each route. That's a follow-up. Until then, this scan is best treated as a regression detector for the unauthenticated shell, not as a coverage-complete audit.
